Resumen de los requisitos de Accesibilidad (WCAG 2.1)

Principios y pautas (WCAG 2.1)
Principios Pautas Criterios de conformidad Técnicas Suficientes/recomendables
Perceptible
1.1: Proporcionar alternativas textuales
1.2: Proporcionar alternativas para los medios tempodependientes
1.3: Crear contenido que pueda presentarse de diversas formas, manteniendo la estructura
1.4: Facilitar a los usuarios el consumo del contenido, tanto visual como auditivamente
1.1.1 Contenido no textual Nivel A ➡ Todo contenido no textual que se presenta al usuario debe tener una alternativa textual
1.2.4 Subtitulos en directo Nivel AA ➡ Se proporcionarán subtitulos para todo el contenido de audio en directo
1.3.3 Características sensoriales Nivel A ➡ Las instrucciones necesarias para entender y operar el contenido no deberán depender solo de la características sensoriales de los componentes como su forma, tamaño, ubicación vistual, orientación o sonido
1.4.4 Cambio de tamaño del texto Nivel AA ➡ Todo el texto puede ser ajustado sin ayudas técnicas hasta un 200 por cierto, no perdiendo su funcionalidad ni contenido
1.1.1 ➡ Proporcionar altenativas de texto para emojis, emoticonos, arte ASCII y leetspeak
1.2.4 ➡ Crear subtítulos para medios sincronizados en vivo y mantenerlos siempre visibles
1.3.3 ➡ Proporcionar identificación textual de elementos que, de otro modo, dependen únicamente de información sensorial para ser comprendidos.'
1.4.4 ➡ Proporcionar controles en la página web que permitan a los usuarios cambiar incrementalmente el tamaño de todo el texto de la página hasta un 200 por ciento.
Operable
2.1: Proporcionar acceso a toda la funcionalidad mediante el teclado
2.2: Proporcionar a los usuarios el tiempo suficiente para leer y usar contenido
2.3: Evitar diseñar contenido que pueda generar ataques, espasmos o convulsiones
2.4: Proporcionar medios par ayudar a los usuarios a navegar y encontrarse en la web
2.1.1 Teclado Nivel A ➡ Toda la funcionalidad del contenido se puede operar a tráves de una interfaz de teclado, sin requerir una velocidad determinada para cada pulsación individual de las teclas, exceptuando los casos en los que se requiera una entrada que dependa de los movimientos del usuario
2.2.3 Sin tiempo Nivel AAA ➡ El tiempo no es importante a la hora de presentar el contenido, exceptuando los medios multimedia sincronizados no interactivos y los eventos en tiempo real
2.3.2 Tres destellos Nivel AAA ➡ Las páginas web no deberán contener elementos que destellen más de tres veces por segundo
2.4.5 Múltiples vias Nivel AA ➡ Se proporcionará más de un camino para localizar una página web dentro de un cojunto de páginas web, exceptuando cuando dicha página forme parte de un proceso
2.1.1 ➡ Usar fomularios HTML y enlaces
2.2.3 ➡ Permitir a los usuarios completar una actividad sin tener un límite de tiempo
2.3.2 ➡ Garantizar que ningún componente del contenido parpadee más de tres veces en un período de 1 segundo.
2.4.5 ➡ Preever a los usuarios un mapa del sitio web
Comprensible
3.1: Hacer que los contenidos textuales resulten legibles y comprensibles
3.2: Hacer que las páginas web aparezcan y operen de manera pedecible
3.3 - Ayudar a los usuarios a evitar y corregir los errores
3.1.3 Palabras inusuales Nivel A ➡ Se proporcionan mecanismos para identificar las definiciones específicas de palabras o frases
3.2.3 Navegación coherente Nivel AA ➡ Los mecanismos de navegación se deberán mantener casi identicos en múltiples páginas web, apareciendo siempre en un mismo orden cada vez que se repiten, a menos que existan cambios ocasionados por el propio usuario
3.3.3 Sugerencias ante errores Nivel AA ➡ Si existen un error a la hora de introducir datos, se presentarán sugerencias al usuario para realizar su corrección, al menos que esto suponga un riesgo para la seguridad
3.1.3 ➡ Proveer una función que busque una palabra en un diccionario online, devolviendo su definición
3.2.3 ➡ Presentar componentes de forma repetida en el mismo orden cada vez que aparecen
3.3.3 ➡ Identificar un campo requerido con la propiedad "aria-required"
Robusto
4.1: Maximizar la compatibilidad con las aplicaciones de usuarios actuales y futuras
4.1.1 Procesamiento Nivel A ➡ En los contenidos implementados mediante el uso de lenguajes de marcas, los elementos tienen las etiquetas de apertura y cierre completas; los elementos están anidados de acuerdo a sus especificaciones; los elementos no contienen atributos duplicados y los ID son únicos
4.1.2 Nombre, función, valor Nivel A ➡ Para todos los componentes de la interfaz de usuario (incluyendo pero no limitado a: elementos de formulario, enlaces y componentes generados por scripts), el nombre y la función pueden ser determinados por software; los estados, propiedades y valores que pueden ser asignados por el usuario pueden ser especificados por software
4.1.1 ➡ Asegurar que las etiquetas de abre y cierre se usan acorde las especificaciones, y asegurar que los atributos "id" son unicos en el sitio web
4.1.2 ➡ Usar aria-labelledby para proveer un nombre para los controles de interfaz de usuario
